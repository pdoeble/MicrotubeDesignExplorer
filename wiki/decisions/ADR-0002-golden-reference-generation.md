# ADR-0002 — Golden reference generation directly from the MATLAB reference (R2024b)

- **Status:** accepted
- **Date:** 2026-07-10
- **Workstream:** W2 (MATLAB reference)
- **Related plan:** /plans/260710-matlab-inventory-and-golden-data.md

## Context

Golden parity tests (AGENTS §10) need immutable MATLAB-derived reference
datasets. MATLAB R2024b is installed on the development machine
(`C:\Program Files\MATLAB\R2024b`). The reference script keeps its physics in
*local functions* of `source_materials/Waermedurchgang_V10_physical.m`, which are
not callable from outside the script, and `/source_materials` is read-only.

## Decision

1. Golden data are generated by **running the original MATLAB script and its
   verbatim-extracted local functions in MATLAB R2024b** — never by a
   re-implementation.
2. `scripts/golden/generate_golden_references.m`:
   - runs the unmodified reference script (`run(...)`) with
     `WAERME_SKIP_EXPORT=1` and a scratch `WAERME_EXPORT_DIR` so nothing is
     written into `/source_materials`;
   - harvests the raw workspace fields (grids, masks, screens, boundaries,
     scalars) after the run;
   - additionally extracts the physics local functions **verbatim by text
     slicing** of the reference file into a scratch folder, and calls them
     directly to produce function-level references across laminar/transition/
     turbulent branches and boundary cases that the fixed-parameter script
     does not visit.
3. Storage format under `/reference`:
   - 2D float64 grids as little-endian binary `.f64` files (C order),
   - scalars, vectors, and metadata as JSON,
   - a `manifest.json` with shapes, SHA-256 hashes, MATLAB version, script
     SHA-256, generation date, and the exact generation command.
4. Golden files are immutable; regeneration requires provenance notes and a
   review of the MATLAB reference (AGENTS §10).

## Consequences

- Parity tests compare against true MATLAB output at `rtol=1e-8`,
  `atol=1e-10`.
- Golden regeneration requires a machine with MATLAB R2024b; the procedure is
  documented in `wiki/model/golden-data.md`.
- Binary grids (~12 MB total) are committed once and treated as immutable.

## Evidence

- Generation log and hashes recorded in `reference/manifest.json`.
